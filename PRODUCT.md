# Product

## Register

product

## Users

Solo power user (the developer). Context: at a desk, working through email volume, running LLM tools locally. No onboarding needed. High tolerance for density; expects every panel and control to earn its place. Using the app beside other tools in a narrow window or focused alongside a browser.

## Product Purpose

A local-first email assistant that syncs IMAP mail, generates AI draft replies, and builds a Markdown knowledge base about contacts. Never sends email. Exists to give the user a private, offline-capable workspace for reading and responding to mail with full LLM assistance, without data leaving the machine.

## Brand Personality

Dense, precise, local. Information-forward. Feels like it runs on your machine, not in a cloud. Obsidian-adjacent: markdown-native, knowledge-graph sensibilities, slightly technical.

## Anti-references

- Generic SaaS (cream backgrounds, rounded-everything, purple/indigo accents): too soft, indistinct, cloud-native in feel
- Corporate mail clients (Outlook blue, heavy chrome, cluttered toolbars): wrong era, wrong density pattern
- Overwrought dark mode (neon on black, glow effects, GPU-demo shadows): try-hard, distracts from content

## Design Principles

1. **Data on screen, not behind clicks.** Default to showing more information; collapse only when the user explicitly asks for it.
2. **Local-first materiality.** The UI should feel like it runs on disk. Muted tones, no gradients as decoration, no cloud-SaaS chrome.
3. **Amber as the knowledge accent, not urgency.** The amber accent signals LLM and knowledge-base actions — it is not a warning color and should not be used for error states.
4. **Keyboard as primary, mouse as fallback.** Every core action has a clear affordance; shortcuts and precision interactions are first-class.
5. **Markdown surfaces are canonical.** Knowledge files and notes are raw text; the UI is a reader/editor, not an abstraction over them.

## Accessibility & Inclusion

WCAG AA minimum. Dark and light themes both maintained. Reduced-motion accommodated (no bounce, no decorative animation). No specific color-blindness requirements stated, but amber-on-dark contrast is a known concern to watch.
